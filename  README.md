# ①課題名
小説家誕生日サーチ

## ②課題内容（どんな作品か）
入力した日付が誕生日の「小説家」を一覧表示する。
日本語版 Wikipedia の Web API を使って、指定した日付の「誕生日」セクションから人物データを取得し、その中から「小説家」を含む人だけを抽出してカード形式で表示する。

ユーザーは次のような操作が可能。
・プルダウンで「月」と「日」を選択して検索
・「今日」ボタンで、ワンクリックで今日の日付をセットして検索
・出てきた小説家カードから、Amazonリンク、Wikipediaの詳細ページへジャンプ

## ⑤工夫した点・こだわった点
エラーでページ自体が止まらないように処理を記載したこと
・検索中の UI と操作ロック
　検索開始時に件数表示を「検索中…」に変更
　一覧エリアを一旦クリア
　フォーム（プルダウン・ボタン）を一時的に disabled にする

・検索終了後（成功・失敗問わず）、必ず操作を再び有効化するよう try / catch /finally で制御し、連打や途中の値変更による不整合を防いだ。

## ⑥難しかった点・次回トライしたいこと（又は機能）
◾Wikipedia / Wikidata のデータ構造の理解とパース
Wikipedia のレスポンス構造（parse / text / sections など）を把握するのに時間がかかった。

とくに「誕生日」セクションだけを抜き出すために、一度 prop=sections でセクション一覧を取得し、そこからインデックスを特定して再リクエストする流れがやや難しかった。

◾代表作情報の取得
当初は Wikidata の P800（代表作） を利用して「主な著作」も表示しようとしたが、
実際には代表作が登録されていない人物も多く、安定した表示が難しかった。
今回は無理に詰め込みすぎず、「誕生日と人物情報＋Wikipediaリンク」のシンプルな形にしている。

◾今後の課題として、作品名をテキストから抽出したり、別の API を組み合わせることで、
「代表作」や「おすすめ作品」まで表示できるとより面白くなると感じた。

## ⑦その他
今回のような検索アプリにはWIKI を使用するのが適切ではなかったと思いました。
公式APIではあるけど、構造化された辞書ではなく、
「みんなで編集している百科事典」のため、作品情報として安定して使うの無理があると感じました.
